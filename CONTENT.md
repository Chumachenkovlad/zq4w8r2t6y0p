# Як працювати з контентом

Сайт зібрано на **[Eleventy (11ty)](https://www.11ty.dev/)** — статичному
генераторі. Готовий HTML більше **не редагується руками**: сторінки описуються
у `src/`, а збірка перетворює їх на статичні файли в `_site/`, які й
деплоються на GitHub Pages.

```
src/
  _data/site.yaml      # заголовок хабу, підзаголовок, порядок секцій
  _includes/base.njk   # СПІЛЬНА обгортка всіх сторінок (head, шрифти, noindex, ассети)
  index.njk            # хаб — будується автоматично зі сторінок
  schedule.njk         # ─┐
  kf800.njk            #  ├ сторінки: метадані + контент у front-matter
  basketball.njk       #  │
  skate-plan.njk       # ─┘
  assets/css/*.css     # стилі (по файлу на сторінку + hub.css)
  assets/js/*.js       # рушії інтерактиву (винесені зі сторінок)
```

## Зібрати / переглянути локально

```bash
npm install        # один раз
npm run serve      # локальний сервер з гарячим перезавантаженням
npm run build      # зібрати у _site/
```

Пуш у `main` сам збирає й деплоїть сайт (`.github/workflows/deploy.yml`).

## Додати нову сторінку

1. Створи `src/<назва>.njk` (або `.md`). Front-matter описує і саму сторінку,
   і її картку на хабі:

   ```yaml
   ---
   layout: base.njk
   permalink: /yoga.html          # адреса у вихідному сайті
   title: "Йога · мобільність"     # <title> сторінки
   fonts: "https://fonts.googleapis.com/css2?family=Manrope&display=swap"
   styles:  [/assets/css/yoga.css] # стилі сторінки (необовʼязково)
   scripts: [{ src: /assets/js/yoga.js }]  # скрипти (необовʼязково)
   hub:                            # ← завдяки цьому блоку зʼявиться картка на хабі
     group: training               # одна з груп із site.yaml
     order: 50                     # порядок усередині групи
     title: "Йога · мобільність"   # заголовок картки
     summary: "Короткий опис."
     chips: ["Мобільність", "20 хв"]
     meta: "Відновлення"           # маленький надпис над заголовком
     accent: "#a78bfa"             # колір акценту картки
     featured: false               # true → велика картка на всю ширину
   ---
   <!-- тіло сторінки: HTML/Markdown/Nunjucks -->
   ```

2. За потреби додай `src/assets/css/yoga.css` і `src/assets/js/yoga.js`.
3. Закоміть і запуш у `main`.

Картка на хабі зʼявиться **сама** — окремого маніфесту немає, єдине джерело
правди це front-matter сторінок (колекція `hubPages` у `.eleventy.js`).
Прибрати сторінку з хабу — видали блок `hub:` (або весь файл).

## Групи хабу

Порядок і назви секцій задаються масивом `groups` у `src/_data/site.yaml`.
Поточні: `schedule` (Розклад), `training` (Тренувальні плани). Щоб додати
секцію — додай обʼєкт у `groups` і вкажи його `id` у полі `hub.group`
потрібних сторінок. Порожні групи не показуються.

## Як влаштована спільна обгортка (`base.njk`)

`base.njk` — єдиний шаблон «типового» HTML. Сторінка керує ним через
front-matter:

| ключ | що робить |
|------|-----------|
| `title` | вміст `<title>` |
| `lang` | мова (типово `uk`) |
| `noindex` | `<meta robots noindex>` — **типово `true`** для всіх сторінок |
| `fonts` | URL Google Fonts (preconnect + link додаються самі) |
| `headScripts` | список зовнішніх `<script>` у `<head>` (напр. React CDN) |
| `styles` | список CSS-файлів |
| `scripts` | список `{ src, type?, presets? }` — скрипти в кінці `<body>` |
| `accentBar` | `true` → кольорова смужка зверху |
| `pageData` | обʼєкт → інʼєктиться як `window.__DATA__` для рушіїв |
| `hub` | метадані картки на хабі |

## Патерн «дані + рушій»

Кожна сторінка = **контент (дані)** + **рушій рендеру (JS)**:

- **schedule** — події/нотатки лежать у `pageData` (front-matter), рушій
  `assets/js/schedule.js` будує календар і `.ics`. Редагуєш розклад —
  міняєш YAML, код не чіпаєш.
- **kf800** — React-застосунок; програма тренувань у `assets/js/kf800-data.js`
  (контент), рушій у `assets/js/kf800.jsx`.
- **basketball / skate-plan** — контент (дні/вправи/кроки) у front-matter,
  HTML рендериться шаблоном Nunjucks, а інтерактив (таби, чекбокси, прогрес)
  винесено в `assets/js/*.js`.

## Збереження прогресу

`assets/js/storage.js` дає `window.storage`: якщо платформа надає власне
сховище — використовується воно, інакше fallback на `localStorage`. Тому
прогрес у баскетболі та скейт-плані тепер зберігається і на GitHub Pages.

## Конвенції

- **Мова / стиль:** контентні сторінки — українською, темна тема.
  Нові сторінки бажано тримати в тому ж дусі.
- **Приватність:** `noindex` стоїть на **всіх** сторінках за замовчуванням
  (через `base.njk`), а `robots.txt` забороняє індексацію всього сайту.
- **Самодостатність:** стилі/скрипти сторінки лежать у власних файлах в
  `assets/` — сторінки не залежать одна від одної.
