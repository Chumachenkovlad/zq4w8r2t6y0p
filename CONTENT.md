# Як працювати з контентом

`hub.html` — це **обвʼязка** (індекс) над усіма сторінками. Він не містить
жорстко вшитих посилань: список карток будується з маніфесту `content.json`.
Тому **додати нову сторінку = додати HTML-файл + один запис у JSON**.

## Додати нову сторінку (3 кроки)

1. Поклади готовий самодостатній `*.html` у корінь репозиторію
   (як `kf800.html`, `basketball.html` тощо).
2. Додай запис у масив `items` у `content.json`:

   ```json
   {
     "id": "yoga",                       // унікальний ключ
     "group": "training",               // одна з груп нижче
     "file": "yoga.html",              // ім'я файлу
     "title": "Йога · мобільність",
     "summary": "Короткий опис того, що всередині.",
     "tags": ["Мобільність", "20 хв"], // 0–5 тегів
     "meta": "Відновлення",            // маленький надпис над заголовком
     "accent": "#a78bfa",              // колір акценту картки (hex)
     "featured": false                  // true → велика картка на всю ширину
   }
   ```

3. Закоміть і запуш у `main` — GitHub Pages (`.github/workflows/deploy.yml`)
   задеплоїть автоматично.

Картка зʼявиться у відповідній групі сама. Видалити сторінку з хабу —
просто прибери її запис із `items`.

## Групи

Порядок і назви секцій задаються масивом `groups` у `content.json`.
Поточні: `schedule` (Розклад), `training` (Тренувальні плани), `other` (Інше).
Щоб додати нову секцію — додай обʼєкт у `groups` і вкажи його `id`
у полі `group` потрібних карток. Порожні групи не показуються.

## Конвенції

- **Мова / стиль:** контентні сторінки — українською, темна тема,
  шрифт `Manrope`. Нові сторінки бажано тримати в тому ж дусі.
- **Приватність:** кожна сторінка має `<meta name="robots" content="noindex, nofollow">`,
  а `robots.txt` забороняє індексацію всього сайту. Зберігай це в нових файлах.
- **Самодостатність:** сторінки не залежать одна від одної — увесь CSS/JS
  всередині файлу (або з CDN). Так їх легко переносити й відкривати окремо.
- **Резервний варіант:** у `hub.html` є вбудована копія мінімального маніфесту
  (`FALLBACK`) на випадок, якщо `content.json` не вдалося завантажити
  (наприклад, при відкритті через `file://`). Достатньо тримати `content.json`
  як єдине джерело правди — `FALLBACK` оновлювати не обовʼязково.
